# UAT с пользователем — Day 13

Скрипт прогона. Цель — за 20 минут пройти весь loop: регистрация → проект → 3 чека → /report → Excel.

## Подготовка (до встречи)

- [ ] Сервер запущен, `curl https://<app-domain>/health` → 200
- [ ] Бот @stager_bot отвечает на /start
- [ ] Сгенерил invite-ссылку (через своего бота: `/invite` в активном проекте)
- [ ] Скопировал ссылку, готов отправить пользователю
- [ ] Поставил 3 тестовых чека сбоку на стол (для съёмки)
- [ ] Записываю экран на 20 минут (для портфолио — может пригодиться)

## Скрипт прогона

### 1. Регистрация (1 минута)

- Отправляю пользователю invite-ссылку
- Она тапает → бот пишет «✅ Добро пожаловать»
- ✅ Если ок → дальше
- ❌ Если «инвайт не найден» → проверяю токен в БД

### 2. Создание проекта (2 минуты)

- Прошу: «Введи /newproject»
- Бот → «Как назовём проект?»
- Пользователь: реальное название текущего проекта
- Бот → «Бюджет (число) или /skip:»
- Пользователь: либо число, либо /skip
- Бот → подтверждение → /yes
- ✅ Если ок → дальше
- ❌ Если бот не отвечает → `docker compose logs bot`

### 3. Первый чек — фото (3 минуты)

- Прошу: «Сфоткай первый чек и отправь боту»
- Бот: «📷 принял»
- Через 5-15 сек: карточка с распознанными полями
- Прошу проверить вендор / сумму / категорию
- Если поля верные — тап ✅ Сохранить
- ✅ Если ок → дальше
- ❌ Если карточка не пришла за 30 сек → `docker compose logs worker`
- ❌ Если сумма не распозналась → жмём ✏️ Сумма, вводим вручную

### 4. Второй чек — текстом (1 минута)

- Прошу попробовать `/add`
- Например: `/add 450 расходники малярный скотч`
- Бот → «✅ 450.00 ₽ · расходники\nмалярный скотч»
- ✅ Если ок → дальше

### 5. Третий чек — снова фото (2 минуты)

- Намеренно даю сложный чек: смятый, или нечёткий
- Смотрим как поведёт себя `confidence < 0.6`
- Если бот пометит ⚠️ — пользователь правит вендора через кнопку

### 6. /report (3 минуты)

- Прошу: «Введи /report»
- Бот → сводка по 3 категориям + кнопка «📥 Скачать таблицу»
- Тап → бот шлёт .xlsx
- Пользователь открывает на телефоне / пересылает себе на почту
- ✅ Если таблица читается и суммы верные → ВСЁ РАБОТАЕТ

### 7. Web admin (опционально, 3 минуты)

- Открываем https://<app-domain> на ноутбуке пользователя
- Логин email + пароль
- Пользователь видит свой проект, фильтрует расходы, скачивает Excel
- Если не нужно — пропускаем, пользователь работает только через бот
